> JAVA 란?

- 특징
  - 객체지향 프로그래밍 (상속, 캡슐화, 다형성, 추상화)
  - 자동 메모리 관리 (Garbage Collection)
  - 운영체제에 독립적 but JVM은 운영체제에 종속적 (JVM을 통해 독립적으로 실행, .class 파일을 JIT compiler가 운영체제에 맞게 실행함)
  - 멀티쓰레드 지원 (C, C++과 달리 운영체제의 도움을 받지 않는다.)
    - 구현방식
      - 화이트박스 -> Thread (별도의 클래스 생성)
      - 블랙박스 -> Runnable (익명 객체 및 람다 활용 가능)
  - 동적 로딩 지원 (객체가 필요한 시점에 로딩됨)
- 장점
  - 위에 말했던 것들이 장점이 될 수 있음
  - 많은 레퍼런스와 라이브러리
  - 앉어성(캡슐화, 실행코드배포, 패키징)
  - 높은생산성과 저비용으로 인해 국내에서 자바가 많이 사용됨
- 단점
  - JVM을 통해서 실행되기때문에 상대적으로 느림
  - 표현력이 안좋다 = 다른 언어에 비해 작성해야하는 코드의 길이가 길다.
  - 예외처리가 불편하다
- 왜 우리는 자바를 사용하는가
  - 위에서 말했던 장점 + 웹으로 프로젝트를 진행하는게 많았음 (기업의 ERP 시스템)

- 노드와의 비교
  - 노드는 들어온 요청을 단일 스레드가 처리함
  - 이벤트를 쌓아두고 이벤트 루프가 따로 수행하도록 만듦
  - 노드 서버는 빠른 시간 내 응답과 동작을 가능하게함
  - 작업을 무시하고 지나치는 경우가 있어서 이부분 처리가 필요함

**동시성과 병렬성의 차이**

- 병렬성 (멀티 코어에서 멀티작업이 이루어지는거)



**객체지향 프로그래밍과 함수형 프로그래밍의 차이는?**

- 프로그램이 명령의 수행인지 vs 함수의 계산인지의 차이

**일급 객체란**

- 일급 객체
  - 모든 일급 객체는 변수나 데이터에 담을수 있다
  - 모든 일급 객체는 함수의 파라미터로 전달할 수 있어야한다. (메소드는 이부분을 충족 못시킨것 같음)
  - 모든 일급 객체는 함수의 리턴값으로 사용 할 수 있어야한다.

>  함수형 프로그래밍의 핵심적인 두 아이디어

- 메소드와 람다를 일급값으로 사용
- 가변 공유 상태가 없는 병렬 실행을 이용해서 효율적이고 안전하게 함수나 메소드를 호출할 수 있음 (stateless)
  - 함수 처리에서 다른 무언가에 의존하지 않기 때문

- 결국 이것마저 거쳐가는 과정이고 람다를 통해 깔끔한 코드를 만들어보자
